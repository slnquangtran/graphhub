# Source Context: GraphHub Core

## Overview
This directory contains the entire backend engine for GraphHub: parsing, graph storage, AI/RAG, MCP server, and REST API.

## Service Map

```
src/
├── index.ts                        CLI entry point (index | serve | serve-api | visualize | docs)
├── visualize.ts                    Standalone Mermaid export script
└── services/
    ├── ingestion/
    │   ├── parser.ts               Tree-sitter AST parser → SymbolDefinition[]
    │   └── ingestion-service.ts    Orchestrates parse → store → embed pipeline
    ├── db/
    │   ├── graph-client.ts         KuzuDB singleton wrapper (runCypher, initializeSchema)
    │   └── graph-exporter.ts       Export graph to Mermaid with subgraphs + classDef
    ├── ai/
    │   ├── embedding-service.ts    Local embeddings via Xenova/all-MiniLM-L6-v2
    │   ├── rag-service.ts          Cosine similarity semantic search over Chunk nodes
    │   └── doc-generator.ts        LLM or heuristic-based purpose/strategy generation
    ├── api/
    │   └── server.ts               Express REST API (port 9000)
    └── mcp/
        └── server.ts               MCP server (stdio transport)
```

## Tech Stack

| Concern | Library |
|---|---|
| Parsing | `web-tree-sitter` (WASM) + `tree-sitter-typescript` + `tree-sitter-javascript` |
| Graph DB | `kuzu` (embedded, local) |
| Embeddings | `@xenova/transformers` (Xenova/all-MiniLM-L6-v2, 384-dim) |
| MCP | `@modelcontextprotocol/sdk` |
| HTTP API | `express` + `cors` |

## KuzuDB Schema

```cypher
// Nodes
CREATE NODE TABLE File(path STRING, language STRING, PRIMARY KEY (path))
CREATE NODE TABLE Symbol(id STRING, name STRING, type STRING, kind STRING,
  range STRING, calls STRING[], import_source STRING, import_specifiers STRING[],
  inputs STRING[], outputs STRING[], purpose STRING, strategy STRING)
CREATE NODE TABLE Chunk(id STRING, text STRING, embedding FLOAT[384], PRIMARY KEY (id))
CREATE NODE TABLE FileHash(path STRING, hash STRING, PRIMARY KEY (path))

// Edges
CREATE REL TABLE CONTAINS(FROM File TO Symbol)
CREATE REL TABLE CALLS(FROM Symbol TO Symbol)
CREATE REL TABLE IMPORTS(FROM File TO File, specifiers STRING[])
CREATE REL TABLE DESCRIBES(FROM Chunk TO Symbol)
```

**Symbol fields:**
- `inputs` — function/method parameters (e.g., `["name: string", "age: number"]`)
- `outputs` — return type (e.g., `["Promise<void>"]`)
- `purpose` — what the function does and why (generated by `npm run docs`)
- `strategy` — how it accomplishes its goal (generated by `npm run docs`)

## Data Flow

```
Source file
  → CodeParser.parse()           (Tree-sitter AST → SymbolDefinition[])
  → IngestionService.indexFile() (deduplicated via SHA-256 file hash)
      → GraphClient.runCypher()  (MERGE File, Symbol, Chunk nodes)
      → EmbeddingService         (doc comment → 384-dim float vector on Chunk)
  → resolveImports()             (File IMPORTS File edges)
  → resolveCalls()               (Symbol CALLS Symbol edges)
```

## Incremental Indexing
Files are SHA-256 hashed before indexing. If the hash matches the stored `FileHash` node, the file is skipped. Pass `force=true` to `indexFile()` / `indexFileFallback()` to override.

## Supported Languages

| Language | Parser | Notes |
|---|---|---|
| TypeScript | tree-sitter-typescript | Full symbol extraction |
| TSX | tree-sitter-typescript (tsx grammar) | Full symbol extraction |
| JavaScript | tree-sitter-javascript | Full symbol extraction |
| Python, Go, Rust, Ruby, etc. | Fallback chunker | Text chunks only, no AST symbols |
| Markdown, text, shell | Fallback chunker | Text chunks only |

## MCP Tools

| Tool | Description |
|---|---|
| `query_graph` | Run raw Cypher against the graph |
| `get_file_symbols` | All symbols in a file |
| `semantic_search` | RAG over doc-comment embeddings |
| `get_context` | Callers + callees of a named symbol |
| `impact_analysis` | Blast radius: direct + indirect callers |

## REST API (port 9000)

| Method | Path | Description |
|---|---|---|
| GET | `/api/workspaces` | List indexed workspace roots |
| POST | `/api/index` | Trigger indexing (`{ targetDir }`) |
| GET | `/api/graph?workspace=` | Graph nodes + edges for visualization |
| POST | `/api/search` | Semantic search (`{ query }`) |
| GET | `/api/symbol/:name` | Symbol detail: inputs, outputs, callers, callees, debt |
| GET | `/health` | Health check |

## What Good Looks Like
- Each service is stateless except `GraphClient` (intentional singleton to share one KuzuDB connection).
- `IngestionService` initializes all dependencies in `initialize()` — never call indexing methods without calling `initialize()` first.
- No business logic in `index.ts`; it is a pure CLI dispatcher.

## What to Avoid
- Do not instantiate `GraphClient` directly; always use `GraphClient.getInstance()`.
- Do not call `EmbeddingService.generateEmbedding()` without first calling `initialize()` (or call through `IngestionService.initialize()` which handles it).
- Avoid `any` types — use `SymbolDefinition` and `SearchResult` interfaces.
